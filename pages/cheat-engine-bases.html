Débuter avec Cheat Engine

Introduction à Cheat Engine

Cheat Engine est un outil open-source qui permet d'analyser et de modifier la mémoire des processus en cours d'exécution. C'est un excellent outil pour apprendre les bases du game hacking.

Prérequis

  • Windows 10/11
  • Cheat Engine (dernière version)
  • Un jeu simple pour pratiquer

1. Installation et Configuration

Étapes d'installation :

  1. Téléchargez Cheat Engine depuis le site officiel
  2. Pendant l'installation, décochez les options supplémentaires non nécessaires
  3. Configurez les exceptions dans votre antivirus si nécessaire

2. Interface de Base

Éléments principaux :

  • Sélecteur de processus
  • Zone de recherche de valeurs
  • Liste des résultats
  • Table d'adresses

Fonctions essentielles :

  • Recherche de valeurs exactes
  • Recherche de valeurs inconnues
  • Modification de valeurs
  • Création de pointeurs

3. Premier Scan - Pas à Pas

  1. Lancez votre jeu de test
  2. Ouvrez Cheat Engine et sélectionnez le processus du jeu
  3. Identifiez une valeur à modifier (ex: points de vie)
  4. Entrez la valeur dans Cheat Engine
  5. Cliquez sur "First Scan"
  6. Modifiez la valeur dans le jeu
  7. Entrez la nouvelle valeur dans Cheat Engine
  8. Cliquez sur "Next Scan"
  9. Répétez jusqu'à trouver l'adresse correcte

4. Types de Scan Avancés

Types de recherche :

  • Exact Value : Pour une valeur précise
  • Increased Value : Valeur qui a augmenté
  • Decreased Value : Valeur qui a diminué
  • Changed Value : Valeur qui a changé
  • Unchanged Value : Valeur qui n'a pas changé

5. Utilisation des Pointeurs

Les pointeurs sont essentiels pour trouver des adresses qui changent à chaque lancement du jeu.

Comment créer un pointeur :

  1. Trouvez d'abord l'adresse de la valeur que vous souhaitez modifier
  2. Faites un clic droit sur l'adresse et sélectionnez "Pointer scan for this address"
  3. Configurez la profondeur de scan (généralement 3-5 niveaux)
  4. Lancez le scan et attendez les résultats
  5. Redémarrez le jeu et vérifiez quels pointeurs fonctionnent toujours

Astuce : Pour confirmer un bon pointeur, redémarrez le jeu plusieurs fois et vérifiez s'il pointe toujours vers la bonne valeur.

6. Création de Scripts

Cheat Engine permet d'automatiser des tâches grâce à son langage de script Lua intégré.

Exemple de script simple :

-- Script pour maintenir une valeur constante
function maintienValeur()
  local adresse = 0x12345678  -- Remplacez par votre adresse
  local valeur = 100
  writeInteger(adresse, valeur)
end

-- Exécution toutes les 100ms
createTimer(nil, 100, maintienValeur)

Fonctions Lua utiles :

  • readInteger(adresse) : Lire une valeur entière
  • writeInteger(adresse, valeur) : Écrire une valeur entière
  • readFloat(adresse) : Lire une valeur flottante
  • writeFloat(adresse, valeur) : Écrire une valeur flottante
  • createTimer(nom, intervalle, fonction) : Créer un timer

8. Techniques Avancées

AOB Scan (Array of Bytes)

Permet de trouver une séquence d'octets spécifique dans la mémoire.

Utile pour créer des trainers qui fonctionnent après les mises à jour du jeu.

Exemple : 55 8B EC 83 EC ?? 8B 45 ?? 89

Code Injection

Permet d'injecter votre propre code dans le processus du jeu.

Utilisé pour des modifications complexes qui ne peuvent pas être réalisées par simple modification de valeurs.

Nécessite des connaissances en assembleur.

13. Techniques d'Injection et Hooking

Méthodes d'Injection :

  • DLL Injection : Injecter une DLL dans un processus 
    // Exemple simplifié d'injection DLL
HANDLE hProcess = OpenProcess(PROCESS_ALL_ACCESS, FALSE, targetPID);
LPVOID pDllPath = VirtualAllocEx(hProcess, 0, strlen(dllPath) + 1, MEM_COMMIT, PAGE_READWRITE);
WriteProcessMemory(hProcess, pDllPath, dllPath, strlen(dllPath) + 1, NULL);
HANDLE hThread = CreateRemoteThread(hProcess, NULL, 0, 
                                   (LPTHREAD_START_ROUTINE)GetProcAddress(
                                   GetModuleHandle("kernel32.dll"), "LoadLibraryA"), 
                                   pDllPath, 0, NULL);
  • Manual Mapping : Charger manuellement une DLL sans utiliser LoadLibrary
  • Process Hollowing : Remplacer le code d'un processus légitime
  • Thread Hijacking : Détourner un thread existant

Techniques de Hooking :

  • IAT Hooking : Modifier la table d'importation d'adresses
  • Inline Hooking : Remplacer les premiers octets d'une fonction 
    // Exemple simplifié de hook inline
BYTE originalBytes[5];
BYTE hookBytes[5] = { 0xE9, 0x00, 0x00, 0x00, 0x00 }; // JMP instruction

// Calculer l'adresse relative pour le saut
*(DWORD*)(hookBytes + 1) = (DWORD)((BYTE*)hookFunction - (BYTE*)targetFunction - 5);

// Sauvegarder les octets originaux et installer le hook
memcpy(originalBytes, targetFunction, 5);
WriteProcessMemory(hProcess, targetFunction, hookBytes, 5, NULL);
  • VTable Hooking : Modifier les tables virtuelles des objets C++
  • Trampoline Hooking : Rediriger vers votre code puis revenir au code original

15. Analyse Mémoire Avancée en C++

Techniques d'analyse :

  • Pattern Scanning : Recherche de signatures d'octets 
    // Exemple simplifié de pattern scanning
bool CompareBytes(const BYTE* data, const BYTE* pattern, const char* mask) {
    for (; *mask; ++mask, ++data, ++pattern) {
        if (*mask == 'x' && *data != *pattern)
            return false;
    }
    return true;
}

DWORD FindPattern(DWORD start, DWORD size, const BYTE* pattern, const char* mask) {
    for (DWORD i = 0; i < size; ++i) {
        if (CompareBytes((BYTE*)(start + i), pattern, mask))
            return start + i;
    }
    return 0;
}
  • Memory Dumping : Extraire des régions de mémoire pour analyse
  • Structure Walking : Naviguer dans les structures de données complexes
  • Reverse Engineering : Reconstruire les classes et structures du jeu

Outils d'aide à l'analyse :

  • ReClass.NET : Outil visuel pour reconstruire les structures de classes
  • Cheat Engine : Pour l'analyse initiale avant implémentation en C++
  • Process Hacker : Explorateur de processus avancé
  • PE Explorer : Analyse des fichiers exécutables

17. Considérations Éthiques et Légales

Aspects légaux :

  • Les conditions d'utilisation (ToS) des jeux interdisent généralement toute forme de triche
  • Certaines juridictions ont des lois spécifiques contre la modification de logiciels
  • La vente de cheats peut entraîner des poursuites judiciaires
  • L'accès non autorisé à des serveurs peut être considéré comme du piratage informatique

Pratiques éthiques :

  • Limitez vos expérimentations aux jeux solo ou à des serveurs privés
  • Utilisez vos connaissances pour améliorer la sécurité des jeux
  • Signalez les vulnérabilités aux développeurs (bug bounty)
  • Partagez vos connaissances à des fins éducatives uniquement

Important : Ce tutoriel est fourni uniquement à des fins éducatives. L'utilisation de ces techniques pour tricher dans des jeux en ligne peut entraîner des bannissements, des poursuites judiciaires et nuire à l'expérience des autres joueurs.

18. Anti-Cheats Modernes et Leurs Mécanismes

Principaux anti-cheats et leurs techniques :

  • Easy Anti-Cheat (EAC)
    • Analyse de processus en temps réel
    • Détection de signatures connues
    • Protection contre le debugging
  • BattlEye
    • Analyse de mémoire dynamique
    • Surveillance des drivers
    • Détection de modifications du jeu
  • Vanguard (Riot)
    • Kernel-level protection
    • Démarrage avec le système
    • Analyse proactive des menaces
  • VAC (Valve)
    • Détection basée sur des signatures
    • Analyse différée des infractions
    • Bannissements par vagues

Mécanismes de protection courants :

  • Integrity Checking : Vérification de l'intégrité des fichiers du jeu
  • Code Obfuscation : Obscurcissement du code pour compliquer le reverse engineering
  • VM Detection : Détection des environnements virtualisés
  • Hardware Fingerprinting : Identification unique du matériel pour les bannissements
  • Behavior Analysis : Analyse des comportements suspects (mouvements inhumains, précision parfaite)

Évolution constante : Les anti-cheats sont constamment mis à jour pour contrer les nouvelles techniques de triche. Ce qui fonctionne aujourd'hui pourrait être détecté demain.

19. Reverse Engineering des Protocoles Réseau

Pour les jeux en ligne, comprendre les communications réseau peut être essentiel pour développer certains types de hacks.

Outils d'analyse réseau :

  • Wireshark : Analyseur de paquets réseau
  • Fiddler : Proxy HTTP pour intercepter les communications
  • Charles Proxy : Alternative à Fiddler avec plus de fonctionnalités
  • Burp Suite : Suite d'outils pour tester la sécurité des applications web

Techniques d'analyse :

  • Packet Sniffing : Capturer et analyser les paquets échangés entre le client et le serveur
  • Man-in-the-Middle (MITM) : Intercepter et modifier les communications
  • Protocol Reverse Engineering : Reconstruire le format des messages échangés
  • Encryption Analysis : Comprendre et contourner les mécanismes de chiffrement

Note : Les jeux modernes utilisent souvent des protocoles chiffrés et des techniques anti-tampering pour protéger leurs communications réseau.

Exemple simplifié d'analyse de paquet :

// Exemple de structure de paquet reconstruite
struct PlayerPositionPacket {
    uint32_t packetId;       // Identifiant du type de paquet (ex: 0x03 pour position)
    uint32_t playerId;       // ID du joueur
    float posX, posY, posZ;  // Position dans le monde
    float rotX, rotY;        // Rotation (angles)
    uint8_t isMoving;        // État de mouvement
    uint8_t checksum;        // Somme de contrôle pour vérifier l'intégrité
};

20. Techniques d'Obfuscation pour Protéger vos Hacks

Pour éviter la détection, il est souvent nécessaire d'obfusquer votre code et de masquer son comportement.

Méthodes d'obfuscation du code :

  • String Encryption : Chiffrer les chaînes de caractères sensibles
  • Control Flow Obfuscation : Rendre le flux d'exécution difficile à suivre
  • Dead Code Insertion : Ajouter du code inutile pour confondre l'analyse
  • Instruction Substitution : Remplacer des instructions simples par des équivalents complexes
  • Anti-Disassembly Techniques : Empêcher l'analyse statique du code

Protection contre la détection :

  • Anti-Debugging : Détecter et contrer les tentatives de débogage
  • Anti-VM : Détecter les environnements virtuels utilisés pour l'analyse
  • Timing Checks : Détecter les ralentissements causés par les débogueurs
  • Self-Modifying Code : Code qui se modifie pendant l'exécution

Conseil : Utilisez plusieurs couches de protection. Une seule technique d'obfuscation est rarement suffisante contre des anti-cheats sophistiqués.

23. Glossaire des Termes Techniques

  • Aimbot : Programme qui aide à viser automatiquement les ennemis
  • ESP (Extra Sensory Perception) : Affichage d'informations normalement invisibles
  • Wallhack : Permet de voir à travers les murs
  • Spinbot : Fait tourner rapidement le personnage pour éviter les tirs
  • RCS (Recoil Control System) : Compense automatiquement le recul des armes
  • Triggerbot : Tire automatiquement quand le curseur passe sur un ennemi
  • Bhop (Bunny Hop) : Automatise les sauts pour augmenter la vitesse
  • Skin Changer : Modifie l'apparence des objets dans le jeu
  • VAC (Valve Anti-Cheat) : Système anti-triche de Valve
  • HWID (Hardware ID) : Identifiant unique du matériel utilisé pour les bannissements
  • Offset : Décalage d'une variable en mémoire par rapport à une adresse de base
  • Signature : Séquence d'octets unique identifiant une fonction
  • Hook : Interception d'appels de fonction
  • Jitter : Variation aléatoire ajoutée aux mouvements pour paraître humain
  • Silent Aim : Modification de tir invisible pour les autres joueurs
  • Packet Manipulation : Modification des données envoyées au serveur

⚠️ Pratique Responsable

Pour vous entraîner, utilisez uniquement :

  • Des jeux solo hors-ligne
  • Des environnements de test dédiés
  • Des jeux dont vous possédez une licence légale
  • Des serveurs privés avec autorisation

N'utilisez jamais ces techniques sur des jeux en ligne publics sans autorisation explicite des développeurs.

24. Sécurité et Protection de votre Environnement

Lorsque vous travaillez sur des projets de game hacking, il est crucial de protéger votre environnement de développement et votre identité.

Protection de votre système :

  • Machine virtuelle : Utilisez des VMs pour isoler vos activités de reverse engineering
  • Snapshots : Créez des points de restauration avant de tester du code potentiellement dangereux
  • Pare-feu : Configurez des règles strictes pour contrôler les connexions entrantes et sortantes
  • Analyse de malware : Vérifiez les outils tiers avant de les utiliser (de nombreux "hacks" sont en réalité des malwares)

Protection de votre identité :

  • VPN : Utilisez un service VPN fiable qui ne conserve pas de logs
  • Comptes séparés : Utilisez des comptes différents pour vos activités de développement et de jeu
  • Anonymisation : Évitez de laisser des informations personnelles dans votre code
  • HWID Spoofing : Techniques pour masquer l'identifiant matériel de votre ordinateur

Important : Les anti-cheats modernes collectent de nombreuses informations sur votre système. Même avec des précautions, il existe toujours un risque de détection et de bannissement.
