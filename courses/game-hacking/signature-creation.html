1. Introduction aux Signatures

Qu'est-ce qu'une Signature ?

Une signature est un pattern unique d'octets qui permet d'identifier une fonction ou une zone de code spécifique. Les signatures sont essentielles pour :

  • Localiser des fonctions après les mises à jour
  • Créer des hooks stables
  • Automatiser l'analyse de code
  • Identifier des versions spécifiques de code
Une bonne signature doit être unique et résistante aux mises à jour mineures.

Processus de Création

4. Validation et Tests

Tests de Fiabilité

  1. Vérification de l'unicité
    • Tester sur différentes versions
    • Vérifier les faux positifs
    • Tester les cas limites
  2. Validation de la robustesse
    • Tests après recompilation
    • Tests avec différentes options de compilation
    • Vérification sur différentes architectures
Une signature doit être testée sur plusieurs versions du programme cible.
