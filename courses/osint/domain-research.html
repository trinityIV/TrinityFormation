Recherche de Domaines

Niveau: Intermédiaire Durée: 4h OSINT

Ce cours vous apprend à mener des investigations approfondies sur les domaines web et les infrastructures en ligne. Vous découvrirez comment collecter, analyser et exploiter les informations liées aux domaines pour cartographier l'empreinte numérique d'une organisation ou d'un individu.

1. Introduction

Pourquoi analyser les domaines?

L'analyse de domaines est une composante essentielle de l'OSINT qui permet de :

  • Cartographier l'empreinte numérique d'une organisation
  • Identifier les relations entre différentes entités
  • Découvrir des informations sur l'infrastructure technique
  • Évaluer la posture de sécurité d'une cible
  • Trouver des points d'entrée potentiels pour des tests d'intrusion

Méthodologie d'analyse de domaines

Une approche structurée pour l'analyse de domaines comprend généralement les étapes suivantes :

  1. Collecte d'informations WHOIS
  2. Analyse des enregistrements DNS
  3. Découverte de sous-domaines
  4. Cartographie de l'infrastructure
  5. Identification des technologies utilisées
  6. Analyse historique
  7. Évaluation de la sécurité
  8. Documentation et rapport
Conseil : Documentez systématiquement vos découvertes à chaque étape. Une bonne documentation est essentielle pour une analyse complète et pour éviter de perdre des informations importantes.

2. Analyse WHOIS

Comprendre les données WHOIS

WHOIS est un protocole qui permet d'obtenir des informations sur l'enregistrement d'un nom de domaine. Les données WHOIS peuvent inclure :

  • Nom et coordonnées du propriétaire (si non protégé)
  • Dates d'enregistrement et d'expiration
  • Registrar (entreprise auprès de laquelle le domaine a été enregistré)
  • Serveurs de noms (DNS)
  • Statut du domaine

Outils d'analyse WHOIS

Outil Description Avantages
whois (ligne de commande) Outil en ligne de commande disponible sur Linux/macOS Simple, rapide, peut être utilisé dans des scripts
ICANN Lookup Service officiel de l'ICANN Fiable, informations à jour
DomainTools Service commercial avec fonctionnalités avancées Historique, informations supplémentaires, visualisations
ViewDNS.info Service en ligne avec multiples outils Interface simple, outils complémentaires

Techniques avancées

Au-delà des informations de base, vous pouvez exploiter les données WHOIS de manière plus approfondie :

  • Recherche inverse - Trouver d'autres domaines enregistrés par la même personne/organisation
  • Analyse historique - Examiner les changements dans les enregistrements WHOIS au fil du temps
  • Corrélation d'informations - Lier les données WHOIS à d'autres sources d'information
  • Identification de patterns - Reconnaître des modèles dans les enregistrements de domaines liés
Attention : Depuis l'entrée en vigueur du RGPD en Europe, de nombreuses informations WHOIS sont masquées par défaut. Les services de protection de la vie privée rendent également ces données moins accessibles.

3. Analyse DNS

Comprendre le DNS

Le DNS (Domain Name System) est l'annuaire d'Internet qui traduit les noms de domaine en adresses IP. L'analyse DNS révèle des informations précieuses sur l'infrastructure d'un domaine.

Types d'enregistrements DNS importants

Type Description Utilité pour l'OSINT
A / AAAA Adresse IPv4 / IPv6 d'un domaine Localisation des serveurs, identification d'infrastructure partagée
MX Serveurs de messagerie Identification des services de messagerie utilisés
NS Serveurs de noms Contrôle du domaine, infrastructure DNS
TXT Texte arbitraire Vérification de propriété, SPF, DKIM, informations diverses
CNAME Alias vers un autre domaine Relations entre domaines, services tiers utilisés
SOA Informations d'autorité Configuration DNS, contacts techniques

Outils d'analyse DNS

  • dig / nslookup - Outils en ligne de commande pour interroger les serveurs DNS
  • DNSDumpster - Service en ligne pour cartographier les enregistrements DNS
  • SecurityTrails - Plateforme d'intelligence DNS avec historique
  • Passive Total - Service de RiskIQ pour l'analyse DNS passive
  • MXToolbox - Suite d'outils pour l'analyse DNS et email

Exemple d'analyse DNS avec dig

# Enregistrements A (IPv4)
$ dig example.com A +short
93.184.216.34

# Enregistrements MX (serveurs de messagerie)
$ dig example.com MX +short
0 .

# Enregistrements TXT
$ dig example.com TXT +short
"v=spf1 -all"

# Enregistrements NS (serveurs de noms)
$ dig example.com NS +short
a.iana-servers.net.
b.iana-servers.net.

# Toutes les informations DNS
$ dig example.com ANY
Conseil : Les enregistrements TXT contiennent souvent des informations précieuses comme les configurations SPF, les clés de vérification pour divers services, et parfois même des informations laissées par les administrateurs.

4. Découverte de sous-domaines

Importance des sous-domaines

Les sous-domaines peuvent révéler :

  • Services et applications internes
  • Environnements de développement et de test
  • Applications oubliées ou abandonnées
  • Infrastructure distribuée
  • Acquisitions et partenariats

Techniques de découverte

Techniques passives
  • Certificats SSL/TLS (Transparency Logs)
  • Moteurs de recherche (Google, Bing)
  • Archives DNS
  • Services OSINT spécialisés
  • Sources publiques (GitHub, Pastebin)
Techniques actives
  • Bruteforce de sous-domaines
  • Transferts de zone DNS
  • Énumération DNS
  • Permutations et altérations
  • Vhost discovery

Outils de découverte de sous-domaines

Outil Type Description
Sublist3r Passif/Actif Outil Python qui utilise plusieurs sources et techniques
Amass Passif/Actif Outil complet de cartographie de surface d'attaque réseau
Subfinder Passif Outil Go rapide pour la découverte passive
crt.sh Passif Recherche dans les logs de transparence des certificats
Gobuster Actif Outil de bruteforce rapide écrit en Go

5. Cartographie d'infrastructure

Analyse d'infrastructure

La cartographie d'infrastructure consiste à identifier et comprendre les composants techniques qui soutiennent un domaine :

  • Serveurs et leur localisation
  • Services cloud utilisés
  • CDN (Content Delivery Networks)
  • Load balancers et proxies
  • Pare-feu et WAF (Web Application Firewalls)
  • Relations entre différents systèmes

Techniques de cartographie

Technique Description Outils
Analyse IP Identification des plages d'IP, ASN, géolocalisation IPinfo, MaxMind, BGPView
Traceroute Analyse du chemin réseau vers la cible traceroute, mtr, VisualRoute
Scan de ports Identification des services exposés Nmap, Masscan, Shodan
Analyse d'en-têtes HTTP Identification de serveurs, CDN, WAF curl, Wappalyzer, SecurityHeaders.com
Analyse de certificats SSL Identification de domaines liés, organisations crt.sh, Censys, SSLMate

Exemple d'analyse d'en-têtes HTTP

$ curl -I https://example.com

Visualisation d'infrastructure

La visualisation aide à comprendre les relations complexes entre les différents composants :

  • Maltego - Plateforme de visualisation et d'analyse de liens
  • SpiderFoot - Outil d'automatisation OSINT avec visualisation
  • Gephi - Plateforme open source pour la visualisation de réseaux
  • D3.js - Bibliothèque JavaScript pour créer des visualisations personnalisées
Conseil : Combinez les données de plusieurs sources pour obtenir une image plus complète de l'infrastructure. Les informations partielles de différentes sources peuvent se compléter pour révéler des relations non évidentes.

6. Identification des technologies

Pourquoi identifier les technologies?

L'identification des technologies utilisées par un site web ou une application peut révéler :

  • Vulnérabilités potentielles (versions obsolètes)
  • Pratiques de développement
  • Capacités techniques de l'organisation
  • Relations avec des fournisseurs tiers
  • Points d'entrée pour des tests plus approfondis

Catégories de technologies

Frontend
  • Frameworks JavaScript (React, Angular, Vue)
  • Bibliothèques CSS (Bootstrap, Tailwind)
  • Plugins et widgets
  • Polyfills et shims
Backend
  • Langages (PHP, Python, Ruby, Java)
  • Frameworks (Laravel, Django, Rails)
  • Serveurs web (Apache, Nginx, IIS)
  • Bases de données (MySQL, PostgreSQL)
Infrastructure
  • CDN (Cloudflare, Akamai)
  • Services cloud (AWS, Azure, GCP)
  • WAF (Web Application Firewalls)
  • Systèmes d'exploitation

Outils d'identification

Outil Description Type
Wappalyzer Extension de navigateur qui identifie les technologies web Gratuit (extension)
BuiltWith Service en ligne pour analyser les technologies d'un site Gratuit (limité) Premium
Retire.js Détection de bibliothèques JavaScript vulnérables Open Source
Netcraft Informations sur les serveurs web et l'infrastructure Gratuit

Exemple avec Wappalyzer

Wappalyzer peut identifier des dizaines de catégories de technologies, notamment :

  • CMS (WordPress, Drupal, Joomla)
  • Frameworks JavaScript (React, Angular, Vue)
  • Serveurs web (Apache, Nginx, IIS)
  • Outils d'analyse (Google Analytics, Matomo)
  • CDN (Cloudflare, Akamai, Fastly)
  • Plateformes e-commerce (Shopify, Magento, WooCommerce)
Conseil : Combinez plusieurs outils d'identification pour obtenir une image plus complète. Chaque outil a ses forces et ses faiblesses dans la détection de certaines technologies.

7. Analyse historique

Importance de l'analyse historique

L'analyse historique d'un domaine peut révéler :

  • Évolution du site et de son contenu
  • Changements de propriétaire ou d'organisation
  • Informations supprimées ou modifiées
  • Anciennes vulnérabilités ou configurations
  • Relations historiques avec d'autres entités

Sources d'informations historiques

Source Type d'information Outil/Service
Archives web Captures historiques de pages web Internet Archive (Wayback Machine), archive.today
Historique WHOIS Changements dans les enregistrements de domaine DomainTools, WhoisXML API
Historique DNS Changements dans les enregistrements DNS SecurityTrails, Passive Total
Certificats SSL/TLS Historique des certificats émis crt.sh, Censys

Techniques d'analyse historique

  • Analyse de contenu - Examiner l'évolution du contenu du site
  • Analyse de structure - Observer les changements dans l'architecture du site
  • Analyse de métadonnées - Extraire les métadonnées des versions historiques
  • Analyse comparative - Comparer différentes versions pour identifier les changements
  • Recherche de contenu supprimé - Identifier les informations qui ont été retirées

Exemple avec Wayback Machine

La Wayback Machine de l'Internet Archive permet de :

  • Visualiser les versions historiques d'un site
  • Naviguer dans l'historique via un calendrier
  • Télécharger des captures d'écran ou du contenu
  • Comparer différentes versions
  • Accéder à des fichiers qui ne sont plus disponibles
Attention : Les archives web ne sont pas exhaustives. Certaines pages peuvent ne jamais avoir été archivées, et d'autres peuvent avoir été exclues à la demande du propriétaire du site (via robots.txt ou demande directe).

8. Évaluation de sécurité

Évaluation passive de la sécurité

L'évaluation passive de la sécurité consiste à analyser les informations disponibles publiquement pour identifier des vulnérabilités potentielles sans interagir directement avec les systèmes cibles.

Aspects à évaluer

Configuration
  • En-têtes de sécurité HTTP
  • Configuration SSL/TLS
  • Politiques de sécurité (CSP, CORS)
  • Exposition d'informations sensibles
  • Configurations par défaut
Vulnérabilités connues
  • Versions obsolètes de logiciels
  • CMS et plugins vulnérables
  • Bibliothèques JavaScript obsolètes
  • Vulnérabilités connues dans les services
  • Failles de configuration communes

Outils d'évaluation de sécurité

Outil Description Type
SecurityHeaders.com Analyse des en-têtes de sécurité HTTP Gratuit
SSL Labs Analyse de la configuration SSL/TLS Gratuit
Shodan Recherche de services exposés et vulnérables Gratuit (limité) Premium
Retire.js Détection de bibliothèques JavaScript vulnérables Open Source
Mozilla Observatory Analyse complète de la sécurité web Gratuit

Exemple d'analyse d'en-têtes de sécurité

$ curl -I https://example.com

9. Étude de cas

Étude de cas : Analyse d'une entreprise fictive

Dans cette étude de cas, nous allons analyser le domaine fictif "acme-corp.com" en utilisant les techniques et outils présentés dans ce cours.

1. Analyse WHOIS

Résultats :

  • Domaine enregistré il y a 15 ans
  • Propriétaire : ACME Corporation
  • Contact technique : it@acme-corp.com
  • Registrar : GoDaddy
  • Serveurs de noms : ns1.acme-hosting.com, ns2.acme-hosting.com

Observations : Le domaine est ancien et semble être géré par une équipe IT interne. Les serveurs de noms suggèrent un hébergement personnalisé.

2. Analyse DNS

Résultats :

  • Enregistrements A : 203.0.113.10
  • Enregistrements MX : mail.acme-corp.com (priorité 10)
  • Enregistrements TXT : v=spf1 include:_spf.acme-corp.com ~all
  • Enregistrements CNAME : www.acme-corp.com → acme-corp.com

Observations : L'entreprise gère son propre serveur de messagerie. La configuration SPF indique une protection contre l'usurpation d'emails.

3. Découverte de sous-domaines

Résultats :

  • www.acme-corp.com - Site principal
  • mail.acme-corp.com - Serveur de messagerie
  • webmail.acme-corp.com - Interface webmail
  • intranet.acme-corp.com - Intranet d'entreprise
  • dev.acme-corp.com - Environnement de développement
  • api.acme-corp.com - API publique
  • careers.acme-corp.com - Portail de recrutement
  • legacy.acme-corp.com - Ancien système

Observations : Plusieurs sous-domaines intéressants, notamment un environnement de développement et un système legacy qui pourraient être moins sécurisés.

4. Identification des technologies

Résultats :

  • Serveur web : Apache 2.4.41
  • CMS : WordPress 5.8
  • Plugins : WooCommerce, Contact Form 7, Yoast SEO
  • JavaScript : jQuery 3.5.1, Bootstrap 4.6
  • Analytics : Google Analytics, HotJar
  • CDN : Cloudflare

Observations : Le site principal utilise WordPress avec plusieurs plugins populaires. La version de jQuery est légèrement obsolète.

5. Analyse historique

Résultats :

  • 2010 : Site statique HTML
  • 2015 : Migration vers WordPress
  • 2018 : Ajout d'une boutique en ligne (WooCommerce)
  • 2020 : Refonte majeure du design
  • 2022 : Ajout d'une section "Investisseurs" avec des rapports financiers

Observations : L'entreprise a évolué d'un simple site vitrine à une plateforme e-commerce, puis a ajouté des informations pour les investisseurs, suggérant une croissance significative.

6. Évaluation de sécurité

Résultats :

  • Configuration SSL : A (SSL Labs)
  • En-têtes de sécurité : B (SecurityHeaders.com)
  • Vulnérabilités potentielles : Plugin Contact Form 7 obsolète
  • Exposition d'informations : Divulgation de version WordPress dans le code source
  • Problèmes de configuration : Directory listing activé sur /wp-content/uploads/

Observations : La sécurité globale est bonne, mais quelques problèmes mineurs ont été identifiés, notamment un plugin obsolète et une exposition d'informations.

Conclusion de l'étude de cas

Cette analyse a permis de dresser un portrait complet de l'empreinte numérique d'ACME Corporation :

  • Entreprise établie avec une présence en ligne mature
  • Infrastructure technique relativement bien gérée
  • Quelques points d'attention en matière de sécurité
  • Évolution vers l'e-commerce et communication avec les investisseurs
  • Utilisation de technologies modernes avec quelques composants légèrement obsolètes

Ces informations pourraient être utiles pour diverses finalités : analyse concurrentielle, évaluation de sécurité, due diligence, etc.

10. Exercices pratiques

Exercice 1: Analyse WHOIS et DNS

Choisissez un domaine public (par exemple, une entreprise ou une organisation) et effectuez une analyse WHOIS et DNS complète :

  1. Collectez les informations WHOIS disponibles
  2. Identifiez tous les types d'enregistrements DNS (A, AAAA, MX, TXT, NS, CNAME)
  3. Analysez les serveurs de messagerie et leur configuration
  4. Examinez les enregistrements TXT pour les configurations SPF, DKIM, DMARC
  5. Documentez vos découvertes dans un rapport structuré

Exercice 2: Découverte de sous-domaines

Pour le même domaine ou un autre, effectuez une découverte de sous-domaines :

  1. Utilisez au moins trois outils différents (par exemple, Sublist3r, Amass, crt.sh)
  2. Combinez les résultats et éliminez les doublons
  3. Vérifiez quels sous-domaines sont actifs
  4. Catégorisez les sous-domaines selon leur fonction apparente
  5. Créez une visualisation des relations entre le domaine principal et ses sous-domaines

Exercice 3: Analyse technologique et historique

Approfondissez votre analyse en examinant les technologies et l'historique :

  1. Identifiez les technologies utilisées sur le site principal et quelques sous-domaines
  2. Recherchez des versions historiques du site via la Wayback Machine
  3. Comparez au moins trois versions historiques pour identifier les changements majeurs
  4. Recherchez d'anciennes pages ou fonctionnalités qui ne sont plus accessibles
  5. Créez une chronologie de l'évolution du site
Important : Pour ces exercices, utilisez uniquement des domaines publics et des techniques passives. N'effectuez pas de tests intrusifs sans autorisation explicite.

Comprendre les données WHOIS

                            
WHOIS est un protocole qui permet d'obtenir des informations sur l'enregistrement d'un nom de domaine. Les données WHOIS peuvent inclure :

                            
    
                                
  • Nom et coordonnées du propriétaire (si non protégé)
    • 
                                
  • Dates d'enregistrement et d'expiration
    • 
                                
  • Registrar (entreprise auprès de laquelle le domaine a été enregistré)
    • 
                                
  • Serveurs de noms (DNS)
    • 
                                
  • Statut du domaine
    • 
                            

                            
                            
Outils d'analyse WHOIS

                            

                                

                                    
                                        

                                            Outil
                                            Description
                                            Avantages
                                        

                                    
                                    
                                        

                                            whois (ligne de commande)
                                            Outil en ligne de commande disponible sur Linux/macOS
                                            Simple, rapide, peut être utilisé dans des scripts
                                        

                                        

                                            ICANN Lookup
                                            Service officiel de l'ICANN
                                            Fiable, informations à jour
                                        

                                        

                                            DomainTools
                                            Service commercial avec fonctionnalités avancées
                                            Historique, informations supplémentaires, visualisations
                                        

                                        

                                            ViewDNS.info
                                            Service en ligne avec multiples outils
                                            Interface simple, outils complémentaires
                                        

                                    
                                

                            

                            
                            
Techniques avancées

                            
Au-delà des informations de base, vous pouvez exploiter les données WHOIS de manière plus approfondie :

                            
    
                                
  • Recherche inverse - Trouver d'autres domaines enregistrés par la même personne/organisation
    • 
                                
  • Analyse historique - Examiner les changements dans les enregistrements WHOIS au fil du temps
    • 
                                
  • Corrélation d'informations - Lier les données WHOIS à d'autres sources d'information
    • 
                                
  • Identification de patterns - Reconnaître des modèles dans les enregistrements de domaines liés
    • 
                            

                            
                            

                                 Attention : Depuis l'entrée en vigueur du RGPD en Europe, de nombreuses informations WHOIS sont masquées par défaut. Les services de protection de la vie privée rendent également ces données moins accessibles.

3. Analyse DNS

                    

                        

                            
Comprendre le DNS

                            
Le DNS (Domain Name System) est l'annuaire d'Internet qui traduit les noms de domaine en adresses IP. L'analyse DNS révèle des informations précieuses sur l'infrastructure d'un domaine.

                            
                            
Types d'enregistrements DNS importants

                            

                                

                                    
                                        

                                            Type
                                            Description
                                            Utilité pour l'OSINT
                                        

                                    
                                    
                                        

                                            A / AAAA
                                            Adresse IPv4 / IPv6 d'un domaine
                                            Localisation des serveurs, identification d'infrastructure partagée
                                        

                                        

                                            MX
                                            Serveurs de messagerie
                                            Identification des services de messagerie utilisés
                                        

                                        

                                            NS
                                            Serveurs de noms
                                            Contrôle du domaine, infrastructure DNS
                                        

                                        

                                            TXT
                                            Texte arbitraire
                                            Vérification de propriété, SPF, DKIM, informations diverses
                                        

                                        

                                            CNAME
                                            Alias vers un autre domaine
                                            Relations entre domaines, services tiers utilisés
                                        

                                        

                                            SOA
                                            Informations d'autorité
                                            Configuration DNS, contacts techniques
                                        

                                    
                                

                            

                            
                            
Outils d'analyse DNS

                            
    
                                
  • dig / nslookup - Outils en ligne de commande pour interroger les serveurs DNS
    • 
                                
  • DNSDumpster - Service en ligne pour cartographier les enregistrements DNS
    • 
                                
  • SecurityTrails - Plateforme d'intelligence DNS avec historique
    • 
                                
  • Passive Total - Service de RiskIQ pour l'analyse DNS passive
    • 
                                
  • MXToolbox - Suite d'outils pour l'analyse DNS et email
    • 
                            

                            
                            
Exemple d'analyse DNS avec dig

                            
# Enregistrements A (IPv4)
$ dig example.com A +short
93.184.216.34

# Enregistrements MX (serveurs de messagerie)
$ dig example.com MX +short
0 .

# Enregistrements TXT
$ dig example.com TXT +short
"v=spf1 -all"

# Enregistrements NS (serveurs de noms)
$ dig example.com NS +short
a.iana-servers.net.
b.iana-servers.net.

# Toutes les informations DNS
$ dig example.com ANY


                            
                            

                                 Conseil : Les enregistrements TXT contiennent souvent des informations précieuses comme les configurations SPF, les clés de vérification pour divers services, et parfois même des informations laissées par les administrateurs.

5. Cartographie d'infrastructure

                    

                        

                            
Analyse d'infrastructure

                            
La cartographie d'infrastructure consiste à identifier et comprendre les composants techniques qui soutiennent un domaine :

                            
    
                                
  • Serveurs et leur localisation
    • 
                                
  • Services cloud utilisés
    • 
                                
  • CDN (Content Delivery Networks)
    • 
                                
  • Load balancers et proxies
    • 
                                
  • Pare-feu et WAF (Web Application Firewalls)
    • 
                                
  • Relations entre différents systèmes
    • 
                            

                            
                            
Techniques de cartographie

                            

                                

                                    
                                        

                                            Technique
                                            Description
                                            Outils
                                        

                                    
                                    
                                        

                                            Analyse IP
                                            Identification des plages d'IP, ASN, géolocalisation
                                            IPinfo, MaxMind, BGPView
                                        

                                        

                                            Traceroute
                                            Analyse du chemin réseau vers la cible
                                            traceroute, mtr, VisualRoute
                                        

                                        

                                            Scan de ports
                                            Identification des services exposés
                                            Nmap, Masscan, Shodan
                                        

                                        

                                            Analyse d'en-têtes HTTP
                                            Identification de serveurs, CDN, WAF
                                            curl, Wappalyzer, SecurityHeaders.com
                                        

                                        

                                            Analyse de certificats SSL
                                            Identification de domaines liés, organisations
                                            crt.sh, Censys, SSLMate

$ curl -I https://example.com

Visualisation d'infrastructure

                            
La visualisation aide à comprendre les relations complexes entre les différents composants :

                            
    
                                
  • Maltego - Plateforme de visualisation et d'analyse de liens
    • 
                                
  • SpiderFoot - Outil d'automatisation OSINT avec visualisation
    • 
                                
  • Gephi - Plateforme open source pour la visualisation de réseaux
    • 
                                
  • D3.js - Bibliothèque JavaScript pour créer des visualisations personnalisées
    • 
                            

                            
                            

                                 Conseil : Combinez les données de plusieurs sources pour obtenir une image plus complète de l'infrastructure. Les informations partielles de différentes sources peuvent se compléter pour révéler des relations non évidentes.
                            

                        

                    

                


                

                    
6. Identification des technologies

                    

                        

                            
Pourquoi identifier les technologies?

                            
L'identification des technologies utilisées par un site web ou une application peut révéler :

                            
    
                                
  • Vulnérabilités potentielles (versions obsolètes)
    • 
                                
  • Pratiques de développement
    • 
                                
  • Capacités techniques de l'organisation
    • 
                                
  • Relations avec des fournisseurs tiers
    • 
                                
  • Points d'entrée pour des tests plus approfondis
    • 
                            

                            
                            
Catégories de technologies

                            

                                

                                    

                                        
Frontend

                                        

                                            
    
                                                
  • Frameworks JavaScript (React, Angular, Vue)
    • 
                                                
  • Bibliothèques CSS (Bootstrap, Tailwind)
    • 
                                                
  • Plugins et widgets
    • 
                                                
  • Polyfills et shims
    • 
                                            

                                        

                                    

                                

                                

                                    

                                        
Backend

                                        

                                            
    
                                                
  • Langages (PHP, Python, Ruby, Java)
    • 
                                                
  • Frameworks (Laravel, Django, Rails)
    • 
                                                
  • Serveurs web (Apache, Nginx, IIS)
    • 
                                                
  • Bases de données (MySQL, PostgreSQL)
    •
