Principes de Base de la Sécurité

Niveau: Débutant Durée: 2h Sécurité

Ce cours couvre les principes fondamentaux de la sécurité informatique, y compris la confidentialité, l'intégrité et la disponibilité. Vous apprendrez les bases de la sécurité et comment protéger vos systèmes contre les menaces courantes.

1. Introduction

La sécurité informatique est devenue un enjeu majeur dans notre monde connecté. Ce cours vous donnera les bases nécessaires pour comprendre les principes de sécurité et protéger vos systèmes.

Nous aborderons les concepts fondamentaux, les menaces courantes, et les mécanismes de défense essentiels que tout professionnel de l'informatique devrait connaître.

2. Concepts Fondamentaux

La triade CIA

La sécurité informatique repose sur trois piliers fondamentaux, souvent appelés la triade CIA :

  • Confidentialité - Assurer que l'information n'est accessible qu'aux personnes autorisées
  • Intégrité - Garantir que l'information n'est pas altérée de manière non autorisée
  • Disponibilité - S'assurer que l'information est accessible quand nécessaire

Autres concepts importants

  • Authentification - Vérifier l'identité d'un utilisateur
  • Autorisation - Déterminer les droits d'accès d'un utilisateur
  • Responsabilité - Tracer les actions des utilisateurs
  • Non-répudiation - Empêcher un utilisateur de nier avoir effectué une action

3. Menaces Courantes

Types de menaces

  • Malwares - Logiciels malveillants (virus, vers, chevaux de Troie)
  • Phishing - Tentatives d'obtenir des informations sensibles en se faisant passer pour une entité de confiance
  • Attaques par déni de service (DoS/DDoS) - Rendre un service indisponible
  • Ingénierie sociale - Manipuler des personnes pour obtenir des informations
  • Attaques par force brute - Essayer toutes les combinaisons possibles
Attention : Les menaces évoluent constamment. Il est essentiel de rester informé des nouvelles techniques d'attaque.

4. Mécanismes de Défense

Protections techniques

  • Pare-feu - Filtre le trafic réseau entrant et sortant
  • Antivirus - Détecte et supprime les logiciels malveillants
  • Chiffrement - Protège la confidentialité des données
  • Authentification multi-facteurs - Renforce la sécurité de l'authentification
  • Mises à jour - Corrige les vulnérabilités connues

Défense en profondeur

La défense en profondeur consiste à mettre en place plusieurs couches de sécurité pour protéger un système. Si une couche est compromise, les autres continuent à protéger le système.

Exemple de défense en profondeur
  1. Sécurité physique (accès aux locaux)
  2. Pare-feu périmétrique
  3. Segmentation réseau
  4. Pare-feu hôte
  5. Antivirus
  6. Contrôles d'accès au système d'exploitation
  7. Contrôles d'accès aux applications
  8. Chiffrement des données

5. Bonnes Pratiques

Pour les utilisateurs

  • Utiliser des mots de passe forts et uniques
  • Activer l'authentification à deux facteurs
  • Mettre à jour régulièrement les logiciels
  • Être vigilant face aux tentatives de phishing
  • Sauvegarder régulièrement les données importantes

Pour les administrateurs

  • Appliquer le principe du moindre privilège
  • Mettre en place une politique de sécurité
  • Former les utilisateurs aux bonnes pratiques
  • Surveiller les activités suspectes
  • Planifier la réponse aux incidents
Conseil : La sécurité est un processus continu, pas un état. Restez vigilant et adaptez vos pratiques aux nouvelles menaces.

6. Exercices Pratiques

Exercice 1: Évaluation des risques

Identifiez les risques de sécurité potentiels dans les scénarios suivants :

Scénario 1: Entreprise

Une entreprise de 50 employés utilise :

  • Un réseau Wi-Fi unique pour tous les employés et visiteurs
  • Des ordinateurs partagés dans certains espaces
  • Un système de messagerie cloud
  • Des mots de passe simples et partagés pour certains comptes

Identifiez au moins 5 risques et proposez des solutions.

Exercice 2: Audit de sécurité personnel

Réalisez un audit de sécurité de vos propres pratiques :

  1. Vérifiez la force de vos mots de passe principaux
  2. Identifiez les services où vous n'avez pas activé l'authentification à deux facteurs
  3. Vérifiez les autorisations des applications sur vos appareils
  4. Examinez vos paramètres de confidentialité sur les réseaux sociaux
  5. Vérifiez si vos données ont été compromises sur Have I Been Pwned

Créez un plan d'action pour remédier aux problèmes identifiés.

Exercice 3: Analyse de configuration de pare-feu

Configuration à analyser
# Règles de pare-feu
ALLOW TCP FROM any TO 192.168.1.10 PORT 22
ALLOW TCP FROM any TO 192.168.1.10 PORT 80
ALLOW TCP FROM any TO 192.168.1.10 PORT 443
ALLOW TCP FROM any TO 192.168.1.10 PORT 3389
ALLOW ICMP FROM any TO any
DENY all FROM any TO any

Identifiez les problèmes potentiels dans cette configuration et proposez des améliorations.
