Programmation Sécurisée

Niveau: Intermédiaire Durée: 4h Sécurité

Ce cours vous apprend les principes et pratiques de la programmation sécurisée pour développer des applications robustes face aux menaces modernes. Vous découvrirez comment intégrer la sécurité dès la conception et tout au long du cycle de développement.

1. Introduction

La programmation sécurisée est l'art de développer des applications qui restent fiables même face à des attaques. Plutôt que de traiter la sécurité comme une fonctionnalité à ajouter après le développement, elle doit être intégrée dès la conception.

Important : Le coût de correction d'une faille de sécurité augmente exponentiellement à mesure que l'on avance dans le cycle de développement. Une vulnérabilité découverte en production peut coûter jusqu'à 100 fois plus cher à corriger qu'une vulnérabilité identifiée pendant la phase de conception.

2. Principes de Sécurité

Principes fondamentaux

  • Défense en profondeur - Mettre en place plusieurs couches de sécurité
  • Principe du moindre privilège - Accorder uniquement les droits nécessaires
  • Séparation des préoccupations - Diviser le système en composants distincts
  • Fail secure - En cas d'erreur, privilégier la sécurité
  • Ne jamais faire confiance aux entrées - Toujours valider les données
Principe KISS (Keep It Simple and Secure)

La complexité est l'ennemie de la sécurité. Plus un système est complexe, plus il est difficile de l'analyser et de garantir sa sécurité. Privilégiez des solutions simples et bien comprises.

3. Validation des Entrées

La validation des entrées est la première ligne de défense contre de nombreuses attaques. Toutes les données provenant de sources externes doivent être considérées comme non fiables.

Stratégies de validation

  • Validation par liste blanche - N'accepter que les entrées connues comme sûres
  • Validation par liste noire - Rejeter les entrées connues comme dangereuses
  • Canonicalisation - Convertir les données en format standard avant validation
  • Validation côté serveur - Ne jamais se fier uniquement à la validation côté client
Mauvaise pratique 
// Validation côté client uniquement
function validateForm() {
    var email = document.getElementById("email").value;
    var emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    return emailRegex.test(email);
}
Bonne pratique 
// Validation côté serveur
function validateEmail($email) {
    // Nettoyage
    $email = trim($email);
    
    // Validation par expression régulière
    if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        return false;
    }
    
    // Vérification supplémentaire (ex: domaine valide)
    $domain = substr(strrchr($email, "@"), 1);
    if (!checkdnsrr($domain, "MX")) {
        return false;
    }
    
    return true;
}

4. Authentification et Autorisation

Authentification sécurisée

  • Utiliser des algorithmes de hachage forts (bcrypt, Argon2) pour les mots de passe
  • Implémenter l'authentification multi-facteurs
  • Appliquer des politiques de mots de passe robustes
  • Limiter les tentatives de connexion
  • Utiliser des jetons d'authentification à durée limitée
Exemple de hachage sécurisé 
// PHP - Utilisation de password_hash (basé sur bcrypt)
$hashedPassword = password_hash($password, PASSWORD_DEFAULT, ['cost' => 12]);

// Vérification
if (password_verify($inputPassword, $hashedPassword)) {
    // Authentification réussie
}

Autorisation

L'autorisation détermine ce qu'un utilisateur authentifié est autorisé à faire.

  • Implémenter un contrôle d'accès basé sur les rôles (RBAC)
  • Appliquer le principe du moindre privilège
  • Vérifier les autorisations à chaque action sensible
  • Centraliser la logique d'autorisation

5. Cryptographie

La cryptographie est essentielle pour protéger les données sensibles, mais son implémentation incorrecte peut créer un faux sentiment de sécurité.

Règle d'or : N'implémentez jamais vos propres algorithmes cryptographiques. Utilisez des bibliothèques éprouvées et maintenues.

Bonnes pratiques

  • Utiliser des algorithmes standards (AES, RSA, ECC)
  • Gérer correctement les clés (rotation, stockage sécurisé)
  • Utiliser des vecteurs d'initialisation (IV) uniques
  • Implémenter le chiffrement authentifié (AEAD)
  • Maintenir les bibliothèques cryptographiques à jour
Exemple de chiffrement sécurisé 
# Python - Utilisation de cryptography.io
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os

# Générer une clé
key = AESGCM.generate_key(bit_length=256)

# Chiffrement
aesgcm = AESGCM(key)
nonce = os.urandom(12)  # Nonce unique pour chaque message
ciphertext = aesgcm.encrypt(nonce, plaintext, associated_data)

# Déchiffrement
plaintext = aesgcm.decrypt(nonce, ciphertext, associated_data)

6. Sécurité dans le SDLC

La sécurité doit être intégrée à chaque étape du cycle de développement logiciel (SDLC).

Phase Activités de sécurité
Planification
  • Définir les exigences de sécurité
  • Évaluer les risques
Conception
  • Modélisation des menaces
  • Conception sécurisée
Développement
  • Revue de code
  • Analyse statique
Test
  • Tests de sécurité
  • Analyse dynamique
Déploiement
  • Durcissement des environnements
  • Gestion des secrets
Maintenance
  • Surveillance de sécurité
  • Gestion des vulnérabilités

Outils d'intégration continue

  • SAST (Static Application Security Testing) - Analyse du code source
  • DAST (Dynamic Application Security Testing) - Tests dynamiques
  • SCA (Software Composition Analysis) - Analyse des dépendances
  • IAST (Interactive Application Security Testing) - Tests interactifs
DevSecOps : L'intégration de la sécurité dans les pratiques DevOps permet d'automatiser les contrôles de sécurité tout au long du pipeline de développement et de déploiement.
