1. Introduction à Java

Qu'est-ce que Java?

Java est un langage de programmation orienté objet créé par James Gosling chez Sun Microsystems (maintenant propriété d'Oracle) en 1995. Il a été conçu pour être portable, sécurisé et robuste. Le code Java est compilé en bytecode qui s'exécute sur une machine virtuelle Java (JVM), ce qui permet aux applications Java de fonctionner sur n'importe quelle plateforme disposant d'une JVM.

Caractéristiques principales:

  • Orienté objet - Tout en Java est un objet
  • Indépendant de la plateforme - "Write Once, Run Anywhere" (WORA)
  • Sécurisé - Exécution dans un environnement sandbox
  • Robuste - Gestion stricte des types et vérification à la compilation
  • Multi-thread - Support intégré pour la programmation concurrente
  • Riche bibliothèque standard (JDK)

Applications de Java

Java est utilisé dans de nombreux domaines:

  • Applications d'entreprise
  • Applications mobiles (Android)
  • Applications web
  • Systèmes embarqués
  • Big Data (Hadoop, Spark)
  • Cloud computing
  • Intelligence artificielle et apprentissage automatique
Pourquoi apprendre Java? Java reste l'un des langages les plus demandés sur le marché du travail. Sa portabilité, sa robustesse et son écosystème mature en font un excellent choix pour de nombreux types de projets. De plus, comprendre Java facilite l'apprentissage d'autres langages comme C#, Kotlin ou Scala.

3. Syntaxe de base

Structure d'un programme Java

Un programme Java typique contient les éléments suivants:

  • Package - Déclaration du package (namespace)
  • Imports - Importation de classes et packages
  • Classes - Définition des classes
  • Méthodes - Fonctions à l'intérieur des classes
  • Variables - Champs et variables locales
  • Commentaires - Documentation du code

Exemple commenté:

// Déclaration du package
package com.example.demo;

// Importation de classes
import java.util.Scanner;
import java.time.LocalDate;

/**
 * Classe principale qui démontre la syntaxe de base de Java
 * Ceci est un commentaire de documentation (Javadoc)
 */
public class ExempleSyntaxe {
    // Variable de classe (statique)
    private static final double PI = 3.14159;
    
    // Variable d'instance
    private String nom;
    
    // Constructeur
    public ExempleSyntaxe(String nom) {
        this.nom = nom;
    }
    
    // Méthode principale - point d'entrée du programme
    public static void main(String[] args) {
        // Création d'un objet
        ExempleSyntaxe exemple = new ExempleSyntaxe("Test");
        
        // Appel de méthode
        exemple.afficherInfo();
        
        // Variable locale
        int age = 25;
        
        // Affichage avec concaténation
        System.out.println("Âge: " + age);
    }
    
    // Méthode d'instance
    public void afficherInfo() {
        System.out.println("Nom: " + nom);
        System.out.println("PI: " + PI);
    }
}

Commentaires et conventions de nommage

Les commentaires sont essentiels pour documenter votre code: 

// Commentaire sur une ligne

/* Commentaire
   sur plusieurs
   lignes */
   
/**
 * Commentaire de documentation (Javadoc)
 * @param parametre Description du paramètre
 * @return Description de la valeur de retour
 * @throws Exception Description de l'exception
 */

Conventions de nommage:

  • Classes: PascalCase (ex: MaClasse)
  • Méthodes et variables: camelCase (ex: maMethode, maVariable)
  • Constantes: SNAKE_CASE_MAJUSCULE (ex: MA_CONSTANTE)
  • Packages: minuscules, séparés par des points (ex: com.example.projet)
Note: Ces conventions ne sont pas imposées par le compilateur, mais elles sont largement adoptées dans la communauté Java et font partie des bonnes pratiques.

Hiérarchie des Types en Java:

Types de référence et classes wrapper

En plus des types primitifs, Java offre des types de référence:

Classes wrapper:

Chaque type primitif a une classe wrapper correspondante:

// Types primitifs
int nombre = 42;
boolean estVrai = true;

// Classes wrapper
Integer nombreObj = Integer.valueOf(42);  // Boxing explicite
Boolean estVraiObj = Boolean.valueOf(true);

// Autoboxing (conversion automatique)
Integer autoboxed = 42;  // Le compilateur fait Integer.valueOf(42)
int unboxed = autoboxed;  // Le compilateur fait autoboxed.intValue()

Autres types de référence importants:

// Chaînes de caractères
String texte = "Hello, World!";

// Tableaux
int[] nombres = {1, 2, 3, 4, 5};

// Classes
Date aujourdhui = new Date();

// Énumérations
enum Jour { LUNDI, MARDI, MERCREDI, JEUDI, VENDREDI, SAMEDI, DIMANCHE }
Attention: Les types primitifs sont stockés directement dans la pile (stack), tandis que les types de référence sont stockés dans le tas (heap) avec une référence dans la pile. Cela a des implications sur les performances et le comportement des comparaisons.

Opérateurs de comparaison:

a && b    // ET logique
a || b    // OU logique
!a        // NON logique

Pour les manipulations au niveau des bits:

a += b    // a = a + b

a *= b    // a = a * b
a /= b    // a = a / b
a %= b    // a = a % b
a &= b    // a = a & b
a |= b    // a = a | b
a ^= b    // a = a ^ b
a <<= b   // a = a << b
a >>= b   // a = a >> b
a >>>= b  // a = a >>> b

6. Structures de contrôle

Instructions conditionnelles

Pour exécuter du code en fonction de conditions:

if-else:

if (condition) {
    // Code exécuté si condition est vraie
} else if (autre_condition) {
    // Code exécuté si autre_condition est vraie
} else {
    // Code exécuté si aucune condition n'est vraie
}

Opérateur ternaire:

// condition ? valeur_si_vrai : valeur_si_faux
int max = (a > b) ? a : b;

switch-case:

switch (expression) {
    case valeur1:
        // Code pour valeur1
        break;
    case valeur2:
        // Code pour valeur2
        break;
    default:
        // Code par défaut
}
Attention: N'oubliez pas le break dans chaque case du switch, sinon l'exécution continue aux cases suivantes!

Switch expressions (Java 12+):

// Switch expression avec yield (Java 13+)
int valeur = switch (jour) {
    case LUNDI, MARDI, MERCREDI, JEUDI, VENDREDI -> 1;
    case SAMEDI, DIMANCHE -> {
        System.out.println("Week-end");
        yield 2;
    }
    default -> 0;
};

Boucles

Pour répéter des instructions:

Boucle for:

// for (initialisation; condition; incrémentation)
for (int i = 0; i < 10; i++) {
    System.out.print(i + " ");
}

// for-each (Java 5+)
int[] tableau = {1, 2, 3, 4, 5};
for (int valeur : tableau) {
    System.out.print(valeur + " ");
}

Boucle while:

// Vérifie la condition avant d'exécuter
while (condition) {
    // Code répété tant que condition est vraie
}

Boucle do-while:

// Exécute au moins une fois, puis vérifie la condition
do {
    // Code exécuté au moins une fois
} while (condition);

Contrôle de boucle:

// Sortir d'une boucle
break;

// Passer à l'itération suivante
continue;

// Étiquettes (labels) pour break/continue ciblés
outer: for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (i * j >= 4) {
            break outer; // Sort des deux boucles
        }
    }
}
Conseil: Choisissez la boucle appropriée selon votre besoin. for pour un nombre connu d'itérations, while pour une condition d'arrêt, do-while quand le code doit s'exécuter au moins une fois.

7. Méthodes

Définition et appel de méthodes

Les méthodes permettent de structurer et réutiliser le code:

Syntaxe de base:

// Déclaration
[modificateurs] type_retour nom_methode([paramètres]) {
    // Corps de la méthode
    return valeur;  // Retourne une valeur du type spécifié
}

// Appel
resultat = objet.nom_methode(arg1, arg2);

Exemple:

public class Calculatrice {
    // Méthode statique (de classe)
    public static int additionner(int a, int b) {
        return a + b;
    }
    
    // Méthode d'instance
    public int multiplier(int a, int b) {
        return a * b;
    }
    
    // Méthode sans valeur de retour
    public void afficherResultat(String operation, int resultat) {
        System.out.println(operation + ": " + resultat);
    }
    
    // Utilisation
    public static void main(String[] args) {
        // Appel de méthode statique
        int somme = Calculatrice.additionner(5, 3);
        
        // Création d'objet et appel de méthode d'instance
        Calculatrice calc = new Calculatrice();
        int produit = calc.multiplier(4, 2);
        
        // Appel de méthode void
        calc.afficherResultat("Somme", somme);
        calc.afficherResultat("Produit", produit);
    }
}

Fonctionnalités avancées

Java offre plusieurs fonctionnalités avancées pour les méthodes:

Paramètres variables (varargs):

// Méthode acceptant un nombre variable d'arguments
public static int somme(int... nombres) {
    int total = 0;
    for (int nombre : nombres) {
        total += nombre;
    }
    return total;
}

// Appels
int s1 = somme(1, 2);           // s1 = 3
int s2 = somme(1, 2, 3, 4, 5);  // s2 = 15

Surcharge de méthodes:

// Plusieurs méthodes avec le même nom mais des paramètres différents
public double calculerAire(double rayon) {
    return Math.PI * rayon * rayon;  // Aire d'un cercle
}

public double calculerAire(double longueur, double largeur) {
    return longueur * largeur;  // Aire d'un rectangle
}

Récursion:

// Méthode qui s'appelle elle-même
public static int factorielle(int n) {
    if (n <= 1) return 1;
    return n * factorielle(n - 1);
}
Note: En Java, toutes les méthodes doivent être définies à l'intérieur d'une classe, contrairement à certains autres langages qui permettent des fonctions autonomes.

8. Tableaux et Collections

Tableaux

Les tableaux permettent de stocker plusieurs valeurs du même type:

Déclaration et initialisation:

// Déclaration
int[] nombres;  // Syntaxe préférée
int nombres[];  // Syntaxe alternative

// Allocation
nombres = new int[5];  // Tableau de 5 entiers initialisés à 0

// Déclaration et allocation combinées
int[] nombres = new int[5];

// Initialisation avec des valeurs
int[] nombres = {1, 2, 3, 4, 5};

// Tableau à deux dimensions
int[][] matrice = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};

Accès et manipulation:

// Accès à un élément (indexé à partir de 0)
int premier = nombres[0];
int dernier = nombres[nombres.length - 1];

// Modification d'un élément
nombres[2] = 42;

// Parcours avec for
for (int i = 0; i < nombres.length; i++) {
    System.out.println(nombres[i]);
}

// Parcours avec for-each
for (int nombre : nombres) {
    System.out.println(nombre);
}

Méthodes utiles:

// Copie de tableau
int[] copie = Arrays.copyOf(nombres, nombres.length);

// Tri
Arrays.sort(nombres);

// Recherche binaire (dans un tableau trié)
int index = Arrays.binarySearch(nombres, 42);

// Remplissage
Arrays.fill(nombres, 0);  // Remplit tout le tableau avec 0

// Comparaison
boolean egaux = Arrays.equals(nombres, copie);

Collections

Le framework Collections de Java offre des structures de données plus flexibles que les tableaux:

List

Collection ordonnée qui permet les doublons.

// ArrayList (tableau dynamique)
List liste = new ArrayList<>();
liste.add("un");
liste.add("deux");
liste.add("trois");

// Accès par index
String element = liste.get(1);  // "deux"

// LinkedList (liste chaînée)
List listeChainee = new LinkedList<>();
Set

Collection qui n'accepte pas les doublons.

// HashSet (non ordonné)
Set ensemble = new HashSet<>();
ensemble.add("un");
ensemble.add("deux");
ensemble.add("un");  // Ignoré (doublon)

// TreeSet (trié)
Set ensembleTrie = new TreeSet<>();
Map

Collection de paires clé-valeur.

// HashMap
Map map = new HashMap<>();
map.put("un", 1);
map.put("deux", 2);
map.put("trois", 3);

// Accès par clé
int valeur = map.get("deux");  // 2

// TreeMap (trié par clé)
Map mapTrie = new TreeMap<>();
Queue

Collection pour traiter les éléments en ordre FIFO.

// LinkedList comme Queue
Queue file = new LinkedList<>();
file.offer("un");
file.offer("deux");
file.offer("trois");

// Récupérer et supprimer le premier élément
String premier = file.poll();  // "un"

// PriorityQueue (file à priorité)
Queue filePriorite = new PriorityQueue<>();
Conseil: Utilisez les collections génériques pour bénéficier de la vérification de type à la compilation. Par exemple, List<String> garantit que seules des chaînes peuvent être ajoutées à la liste.

9. Classes et Objets

Définition de classes

Les classes sont les blocs de construction fondamentaux de la programmation orientée objet en Java: 

public class Personne {
    // Attributs (variables d'instance)
    private String nom;
    private int age;
    
    // Attribut statique (variable de classe)
    private static int nombrePersonnes = 0;
    
    // Constante
    public static final int AGE_MAJORITE = 18;
    
    // Constructeur par défaut
    public Personne() {
        this("Inconnu", 0);
    }
    
    // Constructeur avec paramètres
    public Personne(String nom, int age) {
        this.nom = nom;
        this.age = age;
        nombrePersonnes++;
    }
    
    // Méthodes d'accès (getters)
    public String getNom() {
        return nom;
    }
    
    public int getAge() {
        return age;
    }
    
    // Méthodes de modification (setters)
    public void setNom(String nom) {
        this.nom = nom;
    }
    
    public void setAge(int age) {
        if (age >= 0) {
            this.age = age;
        }
    }
    
    // Méthode d'instance
    public boolean estMajeur() {
        return age >= AGE_MAJORITE;
    }
    
    // Méthode statique
    public static int getNombrePersonnes() {
        return nombrePersonnes;
    }
    
    // Redéfinition de méthode de la classe Object
    @Override
    public String toString() {
        return "Personne [nom=" + nom + ", age=" + age + "]";
    }
}

Utilisation de la classe:

// Création d'objets
Personne p1 = new Personne();
Personne p2 = new Personne("Alice", 30);

// Accès aux attributs via les getters
System.out.println(p2.getNom());  // "Alice"
System.out.println(p2.getAge());  // 30

// Modification d'attributs via les setters
p1.setNom("Bob");
p1.setAge(25);

// Appel de méthode d'instance
boolean majeur = p2.estMajeur();  // true

// Accès à une constante de classe
System.out.println(Personne.AGE_MAJORITE);  // 18

// Appel de méthode statique
int nombre = Personne.getNombrePersonnes();  // 2

// Utilisation de toString()
System.out.println(p1);  // "Personne [nom=Bob, age=25]"

Encapsulation et modificateurs d'accès

L'encapsulation est un principe fondamental de la POO qui consiste à regrouper les données et les méthodes qui les manipulent, et à restreindre l'accès direct aux données.

Modificateurs d'accès:

Modificateur Classe Package Sous-classe Partout
private
default (package-private)
protected
public

Autres modificateurs:

  • static - Appartient à la classe plutôt qu'à une instance
  • final - Ne peut pas être modifié (pour les variables) ou redéfini (pour les méthodes)
  • abstract - Doit être implémenté par une sous-classe
  • synchronized - Contrôle l'accès concurrent
  • transient - Exclut de la sérialisation
  • volatile - Indique que la variable peut être modifiée par différents threads
Bonne pratique: Suivez le principe d'encapsulation en déclarant les attributs comme private et en fournissant des méthodes d'accès (getters) et de modification (setters) publiques si nécessaire.

10. Héritage et Polymorphisme

Héritage

L'héritage permet à une classe d'hériter des attributs et méthodes d'une autre classe: 

// Classe de base (superclasse)
public class Animal {
    protected String nom;
    protected int age;
    
    public Animal(String nom, int age) {
        this.nom = nom;
        this.age = age;
    }
    
    public void manger() {
        System.out.println(nom + " mange.");
    }
    
    public void dormir() {
        System.out.println(nom + " dort.");
    }
    
    public String getNom() {
        return nom;
    }
}

// Classe dérivée (sous-classe)
public class Chien extends Animal {
    private String race;
    
    public Chien(String nom, int age, String race) {
        super(nom, age);  // Appel au constructeur de la superclasse
        this.race = race;
    }
    
    // Nouvelle méthode spécifique à Chien
    public void aboyer() {
        System.out.println(nom + " aboie: Woof!");
    }
    
    // Redéfinition (override) d'une méthode de la superclasse
    @Override
    public void manger() {
        System.out.println(nom + " le chien mange des croquettes.");
    }
    
    public String getRace() {
        return race;
    }
}

Utilisation:

// Création d'objets
Animal animal = new Animal("Inconnu", 5);
Chien chien = new Chien("Rex", 3, "Berger Allemand");

// Appel de méthodes
animal.manger();  // "Inconnu mange."
chien.manger();   // "Rex le chien mange des croquettes."
chien.aboyer();   // "Rex aboie: Woof!"

// Polymorphisme
Animal animal2 = new Chien("Max", 2, "Labrador");

3. Syntaxe de base

                    

                        

                            
Structure d'un programme Java

                            

                                Un programme Java typique contient les éléments suivants:
                            

                            
                            
    
                                
  • Package - Déclaration du package (namespace)
    • 
                                
  • Imports - Importation de classes et packages
    • 
                                
  • Classes - Définition des classes
    • 
                                
  • Méthodes - Fonctions à l'intérieur des classes
    • 
                                
  • Variables - Champs et variables locales
    • 
                                
  • Commentaires - Documentation du code
    • 
                            

                            
                            
Exemple commenté:

                            
// Déclaration du package
package com.example.demo;

// Importation de classes
import java.util.Scanner;
import java.time.LocalDate;

/**
 * Classe principale qui démontre la syntaxe de base de Java
 * Ceci est un commentaire de documentation (Javadoc)
 */
public class ExempleSyntaxe {
    // Variable de classe (statique)
    private static final double PI = 3.14159;
    
    // Variable d'instance
    private String nom;
    
    // Constructeur
    public ExempleSyntaxe(String nom) {
        this.nom = nom;
    }
    
    // Méthode principale - point d'entrée du programme
    public static void main(String[] args) {
        // Création d'un objet
        ExempleSyntaxe exemple = new ExempleSyntaxe("Test");
        
        // Appel de méthode
        exemple.afficherInfo();
        
        // Variable locale
        int age = 25;
        
        // Affichage avec concaténation
        System.out.println("Âge: " + age);
    }
    
    // Méthode d'instance
    public void afficherInfo() {
        System.out.println("Nom: " + nom);
        System.out.println("PI: " + PI);
    }
}

                        

                    

                    
                    

                        

                            
Commentaires et conventions de nommage

                            

                                Les commentaires sont essentiels pour documenter votre code:
                            

                            
                            
// Commentaire sur une ligne

/* Commentaire
   sur plusieurs
   lignes */
   
/**
 * Commentaire de documentation (Javadoc)
 * @param parametre Description du paramètre
 * @return Description de la valeur de retour
 * @throws Exception Description de l'exception
 */

                            
                            
Conventions de nommage:

                            
    
                                
  • Classes: PascalCase (ex: MaClasse)
    • 
                                
  • Méthodes et variables: camelCase (ex: maMethode, maVariable)
    • 
                                
  • Constantes: SNAKE_CASE_MAJUSCULE (ex: MA_CONSTANTE)
    • 
                                
  • Packages: minuscules, séparés par des points (ex: com.example.projet)
    • 
                            

                            
                            

                                
                                Note: Ces conventions ne sont pas imposées par le compilateur, mais elles sont largement adoptées dans la communauté Java et font partie des bonnes pratiques.

                                        Type

                                        32 bits

                                        long
