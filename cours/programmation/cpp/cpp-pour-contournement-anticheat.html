C++ pour le Contournement d'Anti-Cheat

Niveau: Avancé Durée: 8 heures Programmation Game Hacking

Ce cours explore l'utilisation avancée du C++ spécifiquement pour comprendre et analyser les systèmes anti-triche dans les jeux vidéo. Vous apprendrez à appliquer les concepts avancés du C++ pour développer des outils d'analyse et comprendre les mécanismes de protection.

Avertissement Éthique et Légal

Les connaissances présentées dans ce cours sont destinées exclusivement à des fins éducatives, de recherche en sécurité et de développement de contre-mesures. L'utilisation de ces techniques pour tricher dans des jeux en ligne est:

  • Généralement contraire aux conditions d'utilisation des jeux
  • Peut entraîner des bannissements permanents de comptes
  • Peut potentiellement violer des lois sur la propriété intellectuelle ou l'accès non autorisé
  • Nuit à l'expérience de jeu des autres joueurs

Nous encourageons une approche éthique et responsable de ces connaissances.

1. Introduction

Pourquoi C++ pour l'Anti-Cheat?

Le C++ est le langage de prédilection pour l'analyse et le développement d'anti-cheat pour plusieurs raisons:

  • Performance - Le C++ offre des performances proches du métal, essentielles pour les opérations sensibles au timing
  • Accès bas niveau - Permet la manipulation directe de la mémoire et des ressources système
  • Compatibilité - La plupart des jeux et anti-cheats sont développés en C++
  • Flexibilité - Supporte à la fois la programmation orientée objet et la programmation procédurale
  • Écosystème - Riche bibliothèque d'outils et de frameworks pour le reverse engineering
Prérequis

Ce cours suppose que vous avez déjà une bonne compréhension des fondamentaux du C++ et des concepts avancés comme les pointeurs, la gestion de mémoire, et les templates. Si ce n'est pas le cas, nous vous recommandons de suivre d'abord nos cours C++ Fondamentaux et C++ Avancé.

Compétences Requises

Pour tirer le meilleur parti de ce cours, vous devriez être familier avec:

Compétences C++
  • Pointeurs et références
  • Gestion de mémoire (new/delete)
  • Classes et héritage
  • Templates et STL
  • Manipulation de bits
  • Programmation multi-thread
Autres connaissances
  • Architecture x86/x64
  • Bases d'assembleur
  • Système d'exploitation Windows
  • Concepts de sécurité informatique
  • Bases du reverse engineering

Structure du cours

Ce cours est structuré pour couvrir progressivement les aspects du C++ pertinents pour l'analyse et le contournement d'anti-cheat:

  1. Nous commencerons par les fondamentaux de l'API Windows en C++
  2. Nous explorerons ensuite les techniques de manipulation de mémoire
  3. Nous aborderons les méthodes avancées comme le hooking et l'injection
  4. Nous terminerons par le développement d'outils d'analyse complets
Conseil

Pratiquez régulièrement les concepts présentés dans un environnement de test sécurisé. La compréhension théorique est importante, mais c'est la pratique qui vous permettra de maîtriser ces techniques complexes.

2. Windows API et C++

L'API Windows est essentielle pour interagir avec les processus et la mémoire sous Windows:

Headers et bibliothèques importantes:

Manipulation de Processus

Techniques pour interagir avec les processus Windows:

Énumération des processus:

// Exemple d'énumération des processus
void EnumerateProcesses() {
    // Créer un snapshot des processus
    HANDLE hSnapshot = CreateToolhelp32Snapshot(TH32CS_SNAPPROCESS, 0);
    if (hSnapshot == INVALID_HANDLE_VALUE) {
        std::cerr << "CreateToolhelp32Snapshot failed: " << GetLastError() << std::endl;
        return;
    }
    
    PROCESSENTRY32 pe32;
    pe32.dwSize = sizeof(PROCESSENTRY32);
    
    // Obtenir le premier processus
    if (!Process32First(hSnapshot, &pe32)) {
        std::cerr << "Process32First failed: " << GetLastError() << std::endl;
        CloseHandle(hSnapshot);
        return;
    }
    
    // Parcourir tous les processus
    do {
        std::cout << "Process ID: " << pe32.th32ProcessID << ", Name: " << pe32.szExeFile << std::endl;
    } while (Process32Next(hSnapshot, &pe32));
    
    CloseHandle(hSnapshot);
}

Obtention d'un handle de processus:

// Exemple d'obtention d'un handle de processus par nom
HANDLE GetProcessHandleByName(const std::string& processName) {
    DWORD pid = 0;
    HANDLE hSnapshot = CreateToolhelp32Snapshot(TH32CS_SNAPPROCESS, 0);
    
    if (hSnapshot == INVALID_HANDLE_VALUE) {
        return NULL;
    }
    
    PROCESSENTRY32 pe32;
    pe32.dwSize = sizeof(PROCESSENTRY32);
    
    if (Process32First(hSnapshot, &pe32)) {
        do {
            if (processName == pe32.szExeFile) {
                pid = pe32.th32ProcessID;
                break;
            }
        } while (Process32Next(hSnapshot, &pe32));
    }
    
    CloseHandle(hSnapshot);
    
    if (pid == 0) {
        return NULL;
    }
    
    // Ouvrir le processus avec les droits nécessaires
    return OpenProcess(
        PROCESS_VM_READ | PROCESS_VM_WRITE | PROCESS_VM_OPERATION | PROCESS_QUERY_INFORMATION,
        FALSE,
        pid
    );
}
Attention: Les anti-cheats modernes surveillent les appels à OpenProcess et peuvent détecter les tentatives d'accès non autorisées. Des techniques plus avancées sont nécessaires pour éviter la détection.

Énumération des Modules

Techniques pour énumérer et analyser les modules chargés dans un processus:

Énumération des modules:

// Exemple d'énumération des modules d'un processus
void EnumerateModules(DWORD processId) {
    // Créer un snapshot des modules
    HANDLE hSnapshot = CreateToolhelp32Snapshot(TH32CS_SNAPMODULE | TH32CS_SNAPMODULE32, processId);
    if (hSnapshot == INVALID_HANDLE_VALUE) {
        std::cerr << "CreateToolhelp32Snapshot failed: " << GetLastError() << std::endl;
        return;
    }
    
    MODULEENTRY32 me32;
    me32.dwSize = sizeof(MODULEENTRY32);
    
    // Obtenir le premier module
    if (!Module32First(hSnapshot, &me32)) {
        std::cerr << "Module32First failed: " << GetLastError() << std::endl;
        CloseHandle(hSnapshot);
        return;
    }
    
    // Parcourir tous les modules
    do {
        std::cout << "Module: " << me32.szModule << std::endl;
        std::cout << "  Base Address: 0x" << std::hex << (uintptr_t)me32.modBaseAddr << std::endl;
        std::cout << "  Size: 0x" << std::hex << me32.modBaseSize << std::endl;
        std::cout << "  Path: " << me32.szExePath << std::endl;
        std::cout << std::dec; // Revenir en décimal
    } while (Module32Next(hSnapshot, &me32));
    
    CloseHandle(hSnapshot);
}

Obtention d'informations sur un module spécifique:

// Exemple d'obtention d'informations sur un module spécifique
MODULEINFO GetModuleInfo(HANDLE hProcess, const std::string& moduleName) {
    MODULEINFO moduleInfo = {0};
    HMODULE hMods[1024];
    DWORD cbNeeded;
    
    // Énumérer tous les modules du processus
    if (EnumProcessModules(hProcess, hMods, sizeof(hMods), &cbNeeded)) {
        for (unsigned int i = 0; i < (cbNeeded / sizeof(HMODULE)); i++) {
            char szModName[MAX_PATH];
            
            // Obtenir le nom du module
            if (GetModuleBaseNameA(hProcess, hMods[i], szModName, sizeof(szModName))) {
                // Vérifier si c'est le module recherché
                if (moduleName == szModName) {
                    // Obtenir les informations du module
                    GetModuleInformation(hProcess, hMods[i], &moduleInfo, sizeof(moduleInfo));
                    break;
                }
            }
        }
    }
    
    return moduleInfo;
}
Astuce

L'énumération des modules est souvent la première étape pour analyser un jeu ou un anti-cheat. Elle permet d'identifier les composants du système et leurs adresses en mémoire, essentielles pour les étapes suivantes.

3. Manipulation de Mémoire

Lecture et Écriture de Mémoire

Les opérations de lecture et d'écriture de mémoire sont fondamentales pour l'analyse d'anti-cheat:

Lecture de mémoire:

// Template pour lire n'importe quel type de données
template<typename T>
T ReadMemory(HANDLE hProcess, uintptr_t address) {
    T value;
    SIZE_T bytesRead;
    
    if (!ReadProcessMemory(hProcess, (LPCVOID)address, &value, sizeof(T), &bytesRead) || bytesRead != sizeof(T)) {
        throw std::runtime_error("Failed to read memory");
    }
    
    return value;
}

// Utilisation
int health = ReadMemory(hProcess, healthAddress);
float position[3] = ReadMemory(hProcess, positionAddress);

Écriture de mémoire:

// Template pour écrire n'importe quel type de données
template<typename T>
void WriteMemory(HANDLE hProcess, uintptr_t address, const T& value) {
    SIZE_T bytesWritten;
    
    if (!WriteProcessMemory(hProcess, (LPVOID)address, &value, sizeof(T), &bytesWritten) || bytesWritten != sizeof(T)) {
        throw std::runtime_error("Failed to write memory");
    }
}

// Utilisation
WriteMemory(hProcess, healthAddress, 100);
float newPosition[3] = {0.0f, 0.0f, 0.0f};
WriteMemory(hProcess, positionAddress, newPosition);
Attention: Les anti-cheats modernes peuvent détecter les appels à ReadProcessMemory et WriteProcessMemory. Des techniques plus avancées sont souvent nécessaires.

Techniques de Scan Mémoire

Pour localiser des valeurs spécifiques en mémoire:

Scan de valeur simple:

// Exemple de scan pour une valeur spécifique
std::vector<uintptr_t> ScanMemoryForValue(HANDLE hProcess, uintptr_t startAddress, 
                                        uintptr_t endAddress, int valueToFind) {
    std::vector<uintptr_t> results;
    const size_t bufferSize = 4096;
    BYTE buffer[bufferSize];
    
    for (uintptr_t currentAddress = startAddress; currentAddress < endAddress; currentAddress += bufferSize) {
        SIZE_T bytesRead;
        SIZE_T bytesToRead = std::min(bufferSize, (size_t)(endAddress - currentAddress));
        
        if (ReadProcessMemory(hProcess, (LPCVOID)currentAddress, buffer, bytesToRead, &bytesRead)) {
            // Parcourir le buffer
            for (size_t i = 0; i <= bytesRead - sizeof(int); i += sizeof(int)) {
                int* value = (int*)(buffer + i);
                
                if (*value == valueToFind) {
                    results.push_back(currentAddress + i);
                }
            }
        }
    }
    
    return results;
}

Scan par pattern (signature):

// Exemple de scan par pattern
std::vector<uintptr_t> PatternScan(HANDLE hProcess, uintptr_t startAddress, 
                                 uintptr_t endAddress, const std::string& pattern, 
                                 const std::string& mask) {
    std::vector<uintptr_t> results;
    const size_t bufferSize = 4096;
    BYTE buffer[bufferSize];
    
    for (uintptr_t currentAddress = startAddress; currentAddress < endAddress; currentAddress += bufferSize) {
        SIZE_T bytesRead;
        SIZE_T bytesToRead = std::min(bufferSize, (size_t)(endAddress - currentAddress));
        
        if (ReadProcessMemory(hProcess, (LPCVOID)currentAddress, buffer, bytesToRead, &bytesRead)) {
            // Parcourir le buffer
            for (size_t i = 0; i <= bytesRead - mask.length(); i++) {
                bool found = true;
                
                for (size_t j = 0; j < mask.length(); j++) {
                    if (mask[j] != '?' && pattern[j] != buffer[i + j]) {
                        found = false;
                        break;
                    }
                }
                
                if (found) {
                    results.push_back(currentAddress + i);
                }
            }
        }
    }
    
    return results;
}

// Utilisation
std::string pattern = "\x55\x8B\xEC\x83\xEC\x10\x53\x56\x57";
std::string mask = "xxxxxxxxx";
auto addresses = PatternScan(hProcess, moduleBase, moduleBase + moduleSize, pattern, mask);

Pointeurs Multi-niveaux

Les pointeurs multi-niveaux sont couramment utilisés dans les jeux pour accéder à des données dynamiques:

Résolution de pointeurs:

// Exemple de résolution de pointeur multi-niveau
template<typename T>
T ReadMultiLevelPointer(HANDLE hProcess, uintptr_t baseAddress, 
                        const std::vector<uintptr_t>& offsets) {
    uintptr_t address = baseAddress;
    
    for (size_t i = 0; i < offsets.size() - 1; i++) {
        address = ReadMemory<uintptr_t>(hProcess, address + offsets[i]);
        
        if (address == 0) {
            throw std::runtime_error("Null pointer encountered");
        }
    }
    
    return ReadMemory<T>(hProcess, address + offsets.back());
}

// Utilisation
std::vector<uintptr_t> offsets = {0x10, 0x20, 0x30, 0x40};
int health = ReadMultiLevelPointer<int>(hProcess, playerBase, offsets);
Conseil

Les pointeurs multi-niveaux sont souvent utilisés pour accéder aux données des entités dans les jeux. Comprendre comment les résoudre est essentiel pour l'analyse d'anti-cheat.

4. Hooking et Détournement

Concepts de Base du Hooking

Le hooking est une technique qui permet d'intercepter des appels de fonction:

  • Inline Hooking - Modification du code de la fonction cible
  • IAT Hooking - Modification de la table d'importation
  • VTable Hooking - Modification des tables virtuelles d'objets
  • Detour Hooking - Redirection de l'exécution vers une fonction personnalisée
Applications

Le hooking permet d'observer et de modifier le comportement d'un programme sans modifier son code source. Dans le contexte des anti-cheats, il peut être utilisé pour:

  • Intercepter les appels aux fonctions de rendu pour implémenter des ESP (Extra Sensory Perception)
  • Modifier les calculs de physique ou de dommages
  • Contourner les vérifications d'intégrité
  • Analyser le fonctionnement interne de l'anti-cheat lui-même

Inline Hooking

L'inline hooking consiste à remplacer les premiers octets d'une fonction par un saut vers notre fonction:

// Structure pour stocker les informations du hook
struct Hook {
    void* targetFunction;
    void* detourFunction;
    BYTE originalBytes[5];
    bool isEnabled;
};

// Fonction pour installer un hook
bool InstallHook(Hook& hook) {
    if (hook.isEnabled) {
        return false;
    }
    
    // Sauvegarder les octets originaux
    memcpy(hook.originalBytes, hook.targetFunction, sizeof(hook.originalBytes));
    
    // Calculer le déplacement relatif pour le saut
    DWORD relativeAddress = (DWORD)hook.detourFunction - (DWORD)hook.targetFunction - 5;
    
    // Modifier les protections mémoire
    DWORD oldProtect;
    if (!VirtualProtect(hook.targetFunction, sizeof(hook.originalBytes), PAGE_EXECUTE_READWRITE, &oldProtect)) {
        return false;
    }
    
    // Écrire l'instruction de saut (E9 = JMP relatif)
    *(BYTE*)hook.targetFunction = 0xE9;
    *(DWORD*)((DWORD)hook.targetFunction + 1) = relativeAddress;
    
    // Restaurer les protections mémoire
    VirtualProtect(hook.targetFunction, sizeof(hook.originalBytes), oldProtect, &oldProtect);
    
    hook.isEnabled = true;
    return true;
}

// Fonction pour désinstaller un hook
bool RemoveHook(Hook& hook) {
    if (!hook.isEnabled) {
        return false;
    }
    
    // Modifier les protections mémoire
    DWORD oldProtect;
    if (!VirtualProtect(hook.targetFunction, sizeof(hook.originalBytes), PAGE_EXECUTE_READWRITE, &oldProtect)) {
        return false;
    }
    
    // Restaurer les octets originaux
    memcpy(hook.targetFunction, hook.originalBytes, sizeof(hook.originalBytes));
    
    // Restaurer les protections mémoire
    VirtualProtect(hook.targetFunction, sizeof(hook.originalBytes), oldProtect, &oldProtect);
    
    hook.isEnabled = false;
    return true;
}

Exemple d'utilisation:

// Fonction originale
typedef int (*OriginalFunction)(int a, int b);
OriginalFunction originalFunc = nullptr;

// Notre fonction de détournement
int DetourFunction(int a, int b) {
    std::cout << "Appel intercepté avec a=" << a << ", b=" << b << std::endl;
    
    // Appeler la fonction originale
    int result = originalFunc(a, b);
    
    std::cout << "Résultat original: " << result << std::endl;
    
    // Modifier le résultat
    return result * 2;
}

// Installation du hook
void SetupHook() {
    // Adresse de la fonction à hooker
    void* targetAddress = GetProcAddress(GetModuleHandle("game.dll"), "CalculateDamage");
    
    // Créer le hook
    Hook damageHook;
    damageHook.targetFunction = targetAddress;
    damageHook.detourFunction = &DetourFunction;
    damageHook.isEnabled = false;
    
    // Sauvegarder un pointeur vers la fonction originale
    // (nécessite un trampoline pour les hooks plus complexes)
    originalFunc = (OriginalFunction)targetAddress;
    
    // Installer le hook
    InstallHook(damageHook);
}

5. Techniques d'Injection

Injection de DLL

L'injection de DLL permet d'exécuter du code personnalisé dans l'espace d'adressage d'un autre processus:

Méthode CreateRemoteThread:

// Exemple d'injection de DLL avec CreateRemoteThread
bool InjectDLL(DWORD processId, const std::string& dllPath) {
    // Ouvrir le processus cible
    HANDLE hProcess = OpenProcess(
        PROCESS_CREATE_THREAD | PROCESS_QUERY_INFORMATION | PROCESS_VM_OPERATION |
        PROCESS_VM_WRITE | PROCESS_VM_READ,
        FALSE, processId
    );
    
    if (!hProcess) {
        std::cerr << "Failed to open process: " << GetLastError() << std::endl;
        return false;
    }
    
    // Allouer de la mémoire dans le processus cible pour le chemin de la DLL
    LPVOID remoteBuffer = VirtualAllocEx(
        hProcess, NULL, dllPath.length() + 1,
        MEM_COMMIT | MEM_RESERVE, PAGE_READWRITE
    );
    
    if (!remoteBuffer) {
        std::cerr << "Failed to allocate memory: " << GetLastError() << std::endl;
        CloseHandle(hProcess);
        return false;
    }
    
    // Écrire le chemin de la DLL dans la mémoire allouée
    if (!WriteProcessMemory(
            hProcess, remoteBuffer, dllPath.c_str(),
            dllPath.length() + 1, NULL
        )) {
        std::cerr << "Failed to write memory: " << GetLastError() << std::endl;
        VirtualFreeEx(hProcess, remoteBuffer, 0, MEM_RELEASE);
        CloseHandle(hProcess);
        return false;
    }
    
    // Obtenir l'adresse de LoadLibraryA
    LPVOID loadLibraryAddr = (LPVOID)GetProcAddress(
        GetModuleHandle("kernel32.dll"), "LoadLibraryA"
    );
    
    if (!loadLibraryAddr) {
        std::cerr << "Failed to get LoadLibraryA address: " << GetLastError() << std::endl;
        VirtualFreeEx(hProcess, remoteBuffer, 0, MEM_RELEASE);
        CloseHandle(hProcess);
        return false;
    }
    
    // Créer un thread distant qui appelle LoadLibraryA avec le chemin de la DLL
    HANDLE hThread = CreateRemoteThread(
        hProcess, NULL, 0,
        (LPTHREAD_START_ROUTINE)loadLibraryAddr,
        remoteBuffer, 0, NULL
    );
    
    if (!hThread) {
        std::cerr << "Failed to create remote thread: " << GetLastError() << std::endl;
        VirtualFreeEx(hProcess, remoteBuffer, 0, MEM_RELEASE);
        CloseHandle(hProcess);
        return false;
    }
    
    // Attendre la fin du thread
    WaitForSingleObject(hThread, INFINITE);
    
    // Nettoyer
    CloseHandle(hThread);
    VirtualFreeEx(hProcess, remoteBuffer, 0, MEM_RELEASE);
    CloseHandle(hProcess);
    
    return true;
}
Détection: Cette méthode est facilement détectable par les anti-cheats modernes qui surveillent les appels à CreateRemoteThread et LoadLibrary.

Techniques d'Injection Avancées

Des méthodes plus sophistiquées pour éviter la détection:

Injection par SetWindowsHookEx:

// Exemple d'injection par SetWindowsHookEx
HHOOK g_hook = NULL;
HINSTANCE g_hInstance = NULL;

// Fonction de hook (dans la DLL à injecter)
LRESULT CALLBACK KeyboardProc(int nCode, WPARAM wParam, LPARAM lParam) {
    // Code à exécuter dans le processus cible
    
    // Appeler le hook suivant dans la chaîne
    return CallNextHookEx(g_hook, nCode, wParam, lParam);
}

// Fonction pour installer le hook (dans l'injecteur)
bool InjectWithHook(DWORD threadId, const std::string& dllPath) {
    // Charger la DLL
    g_hInstance = LoadLibraryA(dllPath.c_str());
    
    if (!g_hInstance) {
        std::cerr << "Failed to load DLL: " << GetLastError() << std::endl;
        return false;
    }
    
    // Obtenir l'adresse de la fonction de hook
    HOOKPROC hookProc = (HOOKPROC)GetProcAddress(g_hInstance, "KeyboardProc");
    
    if (!hookProc) {
        std::cerr << "Failed to get hook procedure: " << GetLastError() << std::endl;
        FreeLibrary(g_hInstance);
        return false;
    }
    
    // Installer le hook
    g_hook = SetWindowsHookEx(WH_KEYBOARD, hookProc, g_hInstance, threadId);
    
    if (!g_hook) {
        std::cerr << "Failed to set hook: " << GetLastError() << std::endl;
        FreeLibrary(g_hInstance);
        return false;
    }
    
    return true;
}

                            Niveau: Avancé
                            Durée: 8 heures
                            Programmation
                            Game Hacking
                        

                        
Ce cours explore l'utilisation avancée du C++ spécifiquement pour comprendre et analyser les systèmes anti-triche dans les jeux vidéo. Vous apprendrez à appliquer les concepts avancés du C++ pour développer des outils d'analyse et comprendre les mécanismes de protection.

Les connaissances présentées dans ce cours sont destinées exclusivement à des fins éducatives, de recherche en sécurité et de développement de contre-mesures. L'utilisation de ces techniques pour tricher dans des jeux en ligne est:

                            
    
                                
  • Généralement contraire aux conditions d'utilisation des jeux

1. Introduction

                    

                        

                            
Pourquoi C++ pour l'Anti-Cheat?

                            
Le C++ est le langage de prédilection pour l'analyse et le développement d'anti-cheat pour plusieurs raisons:

                            
                            
    
                                
  • Performance - Le C++ offre des performances proches du métal, essentielles pour les opérations sensibles au timing
    • 
                                
  • Accès bas niveau - Permet la manipulation directe de la mémoire et des ressources système
    • 
                                
  • Compatibilité - La plupart des jeux et anti-cheats sont développés en C++
    • 
                                
  • Flexibilité - Supporte à la fois la programmation orientée objet et la programmation procédurale
    • 
                                
  • Écosystème - Riche bibliothèque d'outils et de frameworks pour le reverse engineering
    •

Manipulation de Processus

                            
Techniques pour interagir avec les processus Windows:

                            
                            
Énumération des processus:

                            
// Exemple d'énumération des processus
void EnumerateProcesses() {
    // Créer un snapshot des processus
    HANDLE hSnapshot = CreateToolhelp32Snapshot(TH32CS_SNAPPROCESS, 0);
    if (hSnapshot == INVALID_HANDLE_VALUE) {
        std::cerr << "CreateToolhelp32Snapshot failed: " << GetLastError() << std::endl;
        return;
    }
    
    PROCESSENTRY32 pe32;
    pe32.dwSize = sizeof(PROCESSENTRY32);
    
    // Obtenir le premier processus
    if (!Process32First(hSnapshot, &pe32)) {
        std::cerr << "Process32First failed: " << GetLastError() << std::endl;
        CloseHandle(hSnapshot);
        return;
    }
    
    // Parcourir tous les processus
    do {
        std::cout << "Process ID: " << pe32.th32ProcessID << ", Name: " << pe32.szExeFile << std::endl;
    } while (Process32Next(hSnapshot, &pe32));
    
    CloseHandle(hSnapshot);
}

                            
                            
Obtention d'un handle de processus:

                            
// Exemple d'obtention d'un handle de processus par nom
HANDLE GetProcessHandleByName(const std::string& processName) {
    DWORD pid = 0;
    HANDLE hSnapshot = CreateToolhelp32Snapshot(TH32CS_SNAPPROCESS, 0);
    
    if (hSnapshot == INVALID_HANDLE_VALUE) {
        return NULL;
    }
    
    PROCESSENTRY32 pe32;
    pe32.dwSize = sizeof(PROCESSENTRY32);
    
    if (Process32First(hSnapshot, &pe32)) {
        do {
            if (processName == pe32.szExeFile) {
                pid = pe32.th32ProcessID;
                break;
            }
        } while (Process32Next(hSnapshot, &pe32));
    }
    
    CloseHandle(hSnapshot);
    
    if (pid == 0) {
        return NULL;
    }
    
    // Ouvrir le processus avec les droits nécessaires
    return OpenProcess(
        PROCESS_VM_READ | PROCESS_VM_WRITE | PROCESS_VM_OPERATION | PROCESS_QUERY_INFORMATION,
        FALSE,
        pid
    );
}

                            
                            

                                
                                Attention: Les anti-cheats modernes surveillent les appels à OpenProcess et peuvent détecter les tentatives d'accès non autorisées. Des techniques plus avancées sont nécessaires pour éviter la détection.
                            

                        

                    

                    
                    

                        

                            
Énumération des Modules

                            
Techniques pour énumérer et analyser les modules chargés dans un processus:

                            
                            
Énumération des modules:

                            
// Exemple d'énumération des modules d'un processus
void EnumerateModules(DWORD processId) {
    // Créer un snapshot des modules
    HANDLE hSnapshot = CreateToolhelp32Snapshot(TH32CS_SNAPMODULE | TH32CS_SNAPMODULE32, processId);
    if (hSnapshot == INVALID_HANDLE_VALUE) {
        std::cerr << "CreateToolhelp32Snapshot failed: " << GetLastError() << std::endl;
        return;
    }
    
    MODULEENTRY32 me32;
    me32.dwSize = sizeof(MODULEENTRY32);
    
    // Obtenir le premier module
    if (!Module32First(hSnapshot, &me32)) {
        std::cerr << "Module32First failed: " << GetLastError() << std::endl;
        CloseHandle(hSnapshot);
        return;
    }
    
    // Parcourir tous les modules
    do {
        std::cout << "Module: " << me32.szModule << std::endl;
        std::cout << "  Base Address: 0x" << std::hex << (uintptr_t)me32.modBaseAddr << std::endl;
        std::cout << "  Size: 0x" << std::hex << me32.modBaseSize << std::endl;
        std::cout << "  Path: " << me32.szExePath << std::endl;
        std::cout << std::dec; // Revenir en décimal
    } while (Module32Next(hSnapshot, &me32));
    
    CloseHandle(hSnapshot);
}
